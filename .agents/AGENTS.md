# Project Rules & Customizations

We adopt the coding principles defined in [DietrichGebert/ponytail](https://github.com/DietrichGebert/ponytail).

## 🦥 Ponytail Lazy & YAGNI Coding Guidelines

1. **Write Minimal Code:** "The best code is the code you never wrote." Always prefer the shortest, simplest working solution.
2. **YAGNI (You Ain't Gonna Need It):** Never build abstract wrappers or future-proofing abstractions. Focus strictly on current, explicit user requirements.
3. **Prefer Native Standard APIs:** Prioritize modern vanilla JavaScript/TypeScript features and standard runtime capabilities over installing new NPM packages or writing custom helper libraries.
4. **Code Reduction & Refactoring:** Proactively identify and delete unused code, redundant dependencies, or over-engineered boilerplate.
